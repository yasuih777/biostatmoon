Abstract
pyenv + poetryを利用した環境構築

※このページのOSは以下とする。

OS: Windows11 WSL, Ubuntu 22.4

Pythonにおける環境構築

Pythonの開発環境構築として方法は色々あるが、ここではpyenv + poetryを使用する。

  • pyenv: Pythonのバージョンを管理するツール: 公式リンク
  • poetry: Pythonのライブラリを管理するプラットフォーム: 公式リンク

※蛇足だが、別の方法としてAnacondaによる開発構築はこちら

pyenv・poetryのインストール

あらかじめaptをアップデートしておく。

sudo apt update
sudo apt upgrade

pyenv

  1. pyenvの依存関係をインストール
sudo apt install libssl-dev libffi-dev libncurses5-dev zlib1g zlib1g-dev libreadline-dev libbz2-dev libsqlite3-dev make gcc
  1. pyenvのインストール
curl https://pyenv.run | bash
  1. PATH環境変数の設定
echo '' >> ~/.bashrc
echo 'export PYENV_ROOT="$HOME/.pyenv"' >> ~/.bashrc
echo 'export PATH="$PYENV_ROOT/bin:$PATH"' >> ~/.bashrc
echo 'eval "$(pyenv init --path)"' >> ~/.bashrc

WSL再起動後、以下コマンドを実行。バージョン情報がでればOK

pyenv --version

poetry

  1. poetryのインストール
curl -sSL https://install.python-poetry.org | python3 -
poetry --version
  1. 仮想環境作成先の指定

以下のコマンドにより、プロジェクトのディレクトリ配下で仮想環境.venvが作成されるように設定する。

poetry config virtualenvs.in-project true

開発環境構築

pyenv + poetryでの開発環境管理はディレクトリ単位で行う。

プロジェクトの作成

poetryからプロジェクトを作成する

  1. 新しいディレクトリで作成する
poetry new project_name
  1. 元々あるディレクトりに作成する
poetry init

プロジェクト作成後はpyproject.tomlというファイルができる。これにPythonの開発環境に関するあらゆる情報が記録されていく。

Pythonバージョンの変更

pyenvでプロジェクト内でのPythonバージョンを変更する。対象のバージョンがpyenvにない場合はまずインストールする

# Pythonのインストール
pyenv install x.x.x
# プロジェクトディレクトリ内での変更
pyenv local x.x.x

仮想環境のアクティブ化

poetry shell

アクティベートした仮想環境は以下のコマンドで終了できる。

exit

ライブラリのインストール

poetryで必要なライブラリをインストールする

  • パッケージのバージョンを指定する場合はpackages_name==x.x.xのようにする
poetry add packages_name

ライブラリのアンインストール

poetry remove packages_name

開発環境のインストール

poetryで構築した環境はpyproject.tomlに全て自動で保存されていく。これを別のPCで使うには以下のコマンドを実行する。

  • pyenv, poetry加えてPythonのバージョンは改めて準備する必要がある
poetry install
